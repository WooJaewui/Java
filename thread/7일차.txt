 # 보조 쓰레드 종료방법 interrupt( ).

설명.
보조 스레드의 종료를 기다리는데도 한계가 있다.

메소드.
join( ) : 메인쓰레드에서 어떤 하위 쓰레드를 연결하는 것을 의미한다.
interrupt( ) : 메인쓰레드에서 발생시키면 하위쓰레드가 확인 후에 종료시킬 수 있다.

메인 쓰레드.
보조쓰레드.interrupt( ) : 보조쓰레드를 인터럽트 요청한다.

보조 쓰레드.
if( thread.isInterrupted( ) ) { 
	return ; 
} 
메인쓰레드에서 인터럽트 요청이 들어오면 종료시킨다.

보조 쓰레드 주의사항.
interrupted가 아니라 sleep처럼 다른 상태에서 예외처리를 해줘야된다.



 # 쓰레드 우선순위와 데몬쓰레드.

쓰레드를 식별하는 정보들.
ID : 쓰레드의 식별 값.
name : 쓰레드의 이름.
priority : 쓰레드의 우선순위 숫자가 클 수록 우선순위가 높음. ( 0 ~ 10 )
status : 쓰레드의 상태.

priority 설정.
쓰레드변수.setPriority( Thread.MAX_PRIORITY ) ;

데몬쓰레드 설명.
실제적인 로직을 가지고 있지 않고 다른 쓰레드를 도와줄 수 있는 쓰레드.
다른 쓰레드가 모두 종료되면 데몬쓰레드도 같이 종료된다.
대표적인 예 가비지컬렉터.

데몬쓰레드 설정.
쓰레드변수.setDaemon( true ) ;

데몬쓰레드 사용.
보통 priority를 낮게 맞춰놓고 사용한다. ( 로직을 가지고 있지 않기 때문에 )
